Exercises 02 ‚ÄĒ Kalman Filter: 1D Problems

Section A ‚ÄĒ Conceptual Questions (5 questions)

A1. The Kalman gain K is computed as K = P‚ĀĽHŠĶÄ(HP‚ĀĽHŠĶÄ + R)‚ĀĽ¬Ļ. What happens to K (and to the resulting state estimate) when: - (a) The measurement is perfect: R ‚Üí 0 - (b) The measurement is useless: R ‚Üí ‚ąě - (c) We have no prior information at all: P‚ĀĽ ‚Üí ‚ąě

Answer A1 **(a) R ‚Üí 0 (perfect measurement):** 
K = P‚ĀĽHŠĶÄ (HP‚ĀĽHŠĶÄ + 0)‚ĀĽ¬Ļ = P‚ĀĽHŠĶÄ (HP‚ĀĽHŠĶÄ)‚ĀĽ¬Ļ

For 1D with H=1:  K = P‚ĀĽ / P‚ĀĽ = 1

xŐā = xŐā‚ĀĽ + 1 √ó (z - xŐā‚ĀĽ) = z
We fully trust the measurement and ignore the prediction. **(b) R ‚Üí ‚ąě (useless measurement):** 
K = P‚ĀĽHŠĶÄ (HP‚ĀĽHŠĶÄ + ‚ąě)‚ĀĽ¬Ļ ‚Üí 0

xŐā = xŐā‚ĀĽ + 0 √ó (z - xŐā‚ĀĽ) = xŐā‚ĀĽ
We ignore the measurement completely and keep the prediction. **(c) P‚ĀĽ ‚Üí ‚ąě (no prior information):** 
K = ‚ąě √ó (‚ąě + R)‚ĀĽ¬Ļ ‚Üí 1  (‚ąě dominates the denominator)

xŐā = xŐā‚ĀĽ + 1 √ó (z - xŐā‚ĀĽ) = z
Again we fully trust the measurement. This makes sense ‚ÄĒ if we know nothing about position, the first measurement should be taken at face value. **OKS relevance:** When slip detection sets P ‚Üí ‚ąě, the next sensorbar `update()` will have K ‚Üí 1, meaning the sensorbar measurement fully overrides the (now-unknown) predicted pose. This is the recovery mechanism: one good sensorbar hit re-localizes the robot.

A2. After a predict step, the covariance is P‚ĀĽ = 0.09 m¬≤. After an update step using a measurement with noise R = 0.04 m¬≤, what is the posterior covariance P? (Assume 1D, H = 1)

Answer A2 
K = P‚ĀĽ / (P‚ĀĽ + R) = 0.09 / (0.09 + 0.04) = 0.09 / 0.13 ‚Čą 0.692

P = (1 - K) √ó P‚ĀĽ = (1 - 0.692) √ó 0.09 = 0.308 √ó 0.09 ‚Čą 0.0277 m¬≤

ŌÉ_posterior = ‚ąö0.0277 ‚Čą 0.166 m
Compare to the pre-update uncertainty: ŌÉ_prior = ‚ąö0.09 = 0.30 m. The measurement (ŌÉ_R = ‚ąö0.04 = 0.20 m) is more precise than the prediction (0.30 m), so the posterior (0.166 m) is tighter than either source. This is the "sharper than either alone" property from the main notes. **Alternative formula** (equivalent, often faster): 
1/P = 1/P‚ĀĽ + 1/R = 1/0.09 + 1/0.04 = 11.11 + 25.00 = 36.11
P = 1/36.11 ‚Čą 0.0277 m¬≤  ‚úď

A3. Explain in one sentence each the physical meaning of: (a) the innovation, (b) the innovation covariance S = HP‚ĀĽHŠĶÄ + R, (c) the condition when Mahalanobis gating rejects an update.

Answer A3 **(a) Innovation őĹ = z ‚ąí HxŐā‚ĀĽ:** The innovation is the discrepancy between what the sensor actually measured and what the filter predicted the sensor would measure, before applying any correction. **(b) Innovation covariance S = HP‚ĀĽHŠĶÄ + R:** S is the *expected* spread of the innovation ‚ÄĒ it combines prediction uncertainty (HP‚ĀĽHŠĶÄ) with sensor noise (R) to quantify how surprising a given measurement discrepancy would be. **(c) Mahalanobis gating:** A measurement is rejected when its normalized distance from the prediction exceeds a threshold: `őĹŠĶÄ S‚ĀĽ¬Ļ őĹ > Ōá¬≤_threshold`. This means: "the measurement is so far from our prediction that, given our known uncertainty, the probability of this occurring by random noise alone is below the acceptance threshold." **OKS relevance:** `update()` in `` computes the Mahalanobis distance for the sensorbar line offset. If the robot is far off a line (perhaps because it drove into the wrong aisle), the sensorbar reading will be rejected as an outlier rather than catastrophically corrupting the pose estimate.

A4. The OKS shortcut adds the term min(cov_theta, 1e5) * delta_trans¬≤ to the lateral variance at each prediction step. Under what robot motion scenario does this term become the dominant contributor to lateral uncertainty growth?

Answer A4 This term dominates when: 1. `cov_theta` (heading variance) is large ‚ÄĒ e.g., after several missed sensorbar updates that would normally constrain heading, or after a partial slip event that inflated őł uncertainty without hitting the full-slip threshold. 2. `delta_trans` is large ‚ÄĒ the robot is traveling a long distance in a straight line. The combination is exactly: **long aisle travel without sensorbar hits, after heading has already drifted.** This is the classic OKS failure mode where the robot "wanders" laterally in an aisle with sparse painted lines, until covariance grows large enough to trigger a planning failure or a `ERROR_INVALID_STATE`. Numerically: if `cov_theta = 0.01 rad¬≤` (ŌÉ_őł ‚Čą 5.7¬į) and the robot travels 3 m without a sensorbar update, the additional lateral variance is `0.01 √ó 9 = 0.09 m¬≤` (ŌÉ ‚Čą 0.30 m). The fixed `k_trans_lat_noise √ó delta_trans` term contributes only `0.001 √ó 3 = 0.003 m¬≤` by comparison ‚ÄĒ 30√ó smaller.

A5. OKS does not read the pose.covariance field from the /odom ROS message. Why is this a deliberate and reasonable engineering choice, rather than a bug?

Answer A5 The `pose.covariance` field in a ROS odometry message is populated by the odometry publisher (typically the motor controller firmware or a ROS driver node). In practice: 1. **Publishers often lie:** Many wheel encoder drivers either publish zeros, identity matrices, or canned diagonal values that do not reflect actual measurement quality. 2. **Calibrated fixed parameters are more reliable:** A noise model tuned against real robot motion data (e.g., by driving a robot over a known path and measuring actual position error) gives more accurate covariance than trusting the publisher. 3. **Consistency:** OKS operates across multiple robot generations. Using a fixed noise model in the config YAML means the estimator behavior is predictable and version-controlled, rather than dependent on whatever the firmware team decided to publish. **The downside:** If the actual wheel slip characteristics change (e.g., floor material changes, wheel wear), the fixed model will be wrong until someone re-tunes the parameters. This is why floor-specific tuning files exist in the OKS site configuration.

Section C ‚ÄĒ OKS-Specific Calculation Questions

C1 ‚ÄĒ Prediction step with the estimator noise model

Given: - Current P(x,x) = 0.0025 m¬≤ (ŌÉ_x = 0.05 m, a tight estimate after a sensorbar hit) - P(őł,őł) = 0.0010 rad¬≤ (ŌÉ_őł ‚Čą 1.8¬į, well-constrained heading) - Robot moves forward delta_trans = 0.5 m in this time step - delta_rot = 0.0 rad (straight-line motion) - OKS noise parameters: - k_trans_noise = 0.01 - k_trans_lat_noise = 0.005 - k_time_trans_noise = 0.0 (ignore time component) - No measurement update this step

Question: What is P(x,x) after the prediction step?

(Assume the robot is driving along the x-axis so “longitudinal” ‚Čą x and “lateral” ‚Čą y.)

Answer C1 Using the the estimator noise model from `predict()`: 
őĒP(x,x) = k_trans_noise √ó delta_trans
         + min(P_őłőł, 1e5) √ó delta_trans¬≤

         = 0.01 √ó 0.5
         + min(0.0010, 1e5) √ó 0.5¬≤

         = 0.005
         + 0.0010 √ó 0.25

         = 0.005 + 0.00025

         = 0.00525 m¬≤

P(x,x)_new = P(x,x)_old + őĒP(x,x)
           = 0.0025 + 0.00525
           = 0.00775 m¬≤

ŌÉ_x_new = ‚ąö0.00775 ‚Čą 0.088 m
**Breakdown of contributions:** - Fixed translational noise (`0.01 √ó 0.5 = 0.005`): contributes **95.2%** of new variance - Heading-coupled term (`0.0010 √ó 0.25 = 0.00025`): contributes **4.8%** When P_őłőł is small (well-constrained heading), the fixed noise dominates. The heading-coupled term only becomes dominant when `P_őłőł > k_trans_noise = 0.01 rad¬≤` (ŌÉ_őł > 5.7¬į). In this example, `P_őłőł = 0.001 << 0.01`, so heading uncertainty is not yet the dominant source of error. **ŌÉ_x grew from 0.050 m to 0.088 m** for a 0.5 m move ‚ÄĒ about a 76% increase in standard deviation.

C2 ‚ÄĒ Minimum covariance ratio for K > 0.5

Question: For a 1D filter with measurement matrix H = [1] (direct position measurement), what is the minimum ratio P‚ĀĽ / R such that the Kalman gain K > 0.5?

Derive algebraically and verify with a numerical example using R = 0.04 m¬≤ (ŌÉ = 0.2 m, typical sensorbar uncertainty).

Answer C2 **Algebraic derivation:** 
K = P‚ĀĽ HŠĶÄ (H P‚ĀĽ HŠĶÄ + R)‚ĀĽ¬Ļ

For H = 1 (1D):
K = P‚ĀĽ / (P‚ĀĽ + R)

Set K > 0.5:
P‚ĀĽ / (P‚ĀĽ + R) > 0.5

P‚ĀĽ > 0.5 √ó (P‚ĀĽ + R)
P‚ĀĽ > 0.5 P‚ĀĽ + 0.5 R
0.5 P‚ĀĽ > 0.5 R
P‚ĀĽ > R
**Result: K > 0.5 if and only if P‚ĀĽ > R** Equivalently: the Kalman gain exceeds 0.5 (meaning we trust the sensor more than the prediction) when the prediction uncertainty is larger than the measurement noise. **Numerical verification with R = 0.04 m¬≤ (ŌÉ_sensor = 0.2 m):** 
P‚ĀĽ = 0.03 m¬≤  (< R):  K = 0.03 / (0.03 + 0.04) = 0.03/0.07 = 0.429 < 0.5  ‚úď
P‚ĀĽ = 0.04 m¬≤  (= R):  K = 0.04 / 0.08 = 0.500 = 0.5 (boundary)             ‚úď
P‚ĀĽ = 0.05 m¬≤  (> R):  K = 0.05 / 0.09 = 0.556 > 0.5                        ‚úď
P‚ĀĽ = 0.10 m¬≤  (> R):  K = 0.10 / 0.14 = 0.714 > 0.5                        ‚úď
**OKS interpretation:** - `R = 0.04 m¬≤` is typical for a well-calibrated sensorbar (ŌÉ ‚Čą 0.2 m lateral accuracy) - `P(x,x) = 0.04 m¬≤` means the robot has accumulated uncertainty equivalent to the sensor noise - For `P(x,x) > 0.04 m¬≤`, the filter trusts the sensorbar more than its dead-reckoning prediction In practice, after a robot travels ~4 m without a sensorbar update (őĒP_xx ‚Čą 0.01 √ó 4 = 0.04 m¬≤ from `k_trans_noise`), the transition point K = 0.5 is reached. This is consistent with OKS site layout requirements: sensorbar lines should be placed no more than ~4 m apart on heavily-traveled aisles for reliable localization.
