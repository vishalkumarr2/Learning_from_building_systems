Why this exists

This site is a living index of lessons learned while solving real-world problems across software, electronics, mechanical, and systems engineering. Use the sidebar to jump to a domain and explore the notes there.

Principles I keep returning to

  • • Debug before you build (logging and testing first).
  • • Systems should be fail-safe and observable.
  • • Mocking makes risky ideas safe to test.
  • • Define the start state and end-of-life state early.
  • • Real systems are non-ideal; set thresholds.
  • • Validate inputs: type, constraints, source.

How to use this site
